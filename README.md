# Madisen's Professional Portfolio

## Purpose

This portfolio website showcases my professional expertise as an **IT Specialist and Data Analyst**. This serves as a central hub for potential employers, collaborators, and professional connections to learn about my work, skills, and approach to solving complex technical problems.

## Key Objectives

- **Demonstrate Technical Expertise**: Display real-world projects in IT operations, data analysis, systems administration, and web development
- **Professional Visibility**: Maintain an up-to-date online presence across multiple professional networks
- **Facilitate Connections**: Provide easy ways for recruiters, colleagues, and peers to reach out and collaborate
- **Document Growth**: Track progression through certifications, research interests, and career development

## Structure

The website is organized into five main sections:

### **Home**
- Hero section introducing my role and professional focus
- Quick navigation to explore projects and connect

### **Work & Projects**
Portfolio showcasing key projects including:
- Web Portfolio & Personal Branding
- Exploratoty Data Analysis

### **About**
- Professional background and personality
- Focus on IT operations * data analysis
- Interests in emerging technologies and interdisciplinary research

### **Network**
- **LinkedIn**: IT operations, career development, industry insights
- **EDUCAUSE**: Higher education IT community
- **GitHub**: Code repositories for data analysis, automation, and technical projects
- **Information Sciences Community**: Academic and research-focused networks (ALISE, research forums)

### **Contact**
- Direct email and location information
- Contact form for inquiries
- Social media and professional links

## Technical Stack

- **Frontend**: HTML5, CSS3 (with CSS variables for theming)
- **JavaScript**: Vanilla JS for navigation, form handling, and interactivity
- **Design**: Responsive layout with light/dark mode support
- **Accessibility**: WCAG-compliant color contrasts and semantic HTML
- **Deployment Ready**: Single-file structure with no external dependencies
